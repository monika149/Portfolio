Azure OpenAI Cosmos DB Hackathon

A web application I built with a teammate during a Microsoft hackathon, leveraging Azure OpenAI and Cosmos DB to create a basic AI assistant for productivity and knowledge tasks.

My Hackathon Experience

This was my first time integrating multiple Azure services in a real-world scenario. My teammate focused on backend integration and database management, while I handled the frontend, prompt engineering, and overall user experience. We learned how to quickly prototype, collaborate, and adapt to new APIs under time pressure.

Key Features

Natural Language Q&A

We used Azure OpenAI to enable the assistant to answer user questions in natural language.

Persistent Chat History

Cosmos DB was used to store and retrieve user chat sessions, making the experience seamless and continuous.

Simple, Clean UI

I designed a minimal interface for fast iteration and easy use during the hackathon.

Technical Implementation

How We Built It

  • My teammate handled backend integration with Azure OpenAI and Cosmos DB, ensuring secure API calls and data persistence.
  • I built the frontend (React) and focused on prompt engineering for the LLM, as well as the overall user experience and UI design.
  • We used Azure authentication and managed permissions for secure access.
  • We worked closely to ensure smooth communication between frontend and backend, especially under hackathon time constraints.

Challenges & What I Learned

Rapid Prototyping & Collaboration

We had to quickly divide tasks and trust each other's expertise. I learned a lot about integrating cloud APIs, handling authentication, and designing prompts for best LLM results, while my teammate ensured robust backend logic and data flow.

Future Enhancements

  • Add user authentication and role management
  • Integrate more Microsoft 365 APIs
  • Improve UI/UX and add analytics
